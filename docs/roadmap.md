# 牛马信仰 — Roadmap

> 本文档记录阶段目标和优先级，不替代 `design-doc.md`、`api-contract.md` 或 `data-model.md`。如果 roadmap 与契约文档冲突，以契约文档为准，并同步修正本文件。

## 当前阶段

**阶段：MVP 收敛期**

目标是把已经形成的 Tauri 桌面应用稳定下来：任务生命周期、信仰积分、日历、看板、进程绑定、浏览器 Mock 与真实 Tauri 运行时保持一致。

## 优先级

### P0：稳定现有闭环

- 任务创建、开始、暂停、继续、完成、放弃、删除行为稳定。
- 历史日期保护在前端、后端、Mock 中一致。
- 信仰值、等级、护甲、每日统计与 `data-model.md` / `workflows.md` 一致。
- `main.rs` 与 `tauri/commands.rs` 暴露的命令集保持一致。
- 浏览器模式的 `mock-invoke.ts` 与真实 Tauri 命令契约保持一致。

### P1：完善高频体验

- 日历月/周/日视图的任务可见性、只读状态、今日定位稳定。
- 看板列、卡片拖拽、计时器、任务状态同步可靠。
- 进程绑定在 Windows 环境下可解释、可关闭、失败时不破坏任务状态。
- 悬浮窗与主窗口的状态同步清晰。

### P2：可维护与发布准备

- 建立稳定的视觉回归和核心流程测试清单。
- 补齐构建、恢复、数据迁移和故障排查文档。
- 明确正式发布前的数据备份、迁移和兼容策略。
- 将已经废弃的旧设计持续记录在 `ai-collaboration.md` 或 `decisions.md`。

## 非目标

- 不做云端账号系统。
- 不做多人协作或社区功能，除非需求文档明确进入新阶段。
- 不把浏览器 Mock 当成独立产品形态。
- 不引入新的状态管理、数据库、UI 框架，除非已有架构无法支撑明确需求。

## 里程碑

| Milestone | 目标 | 完成标准 |
|-----------|------|----------|
| M1 稳定任务闭环 | 任务和信仰积分核心路径可靠 | P0 项通过构建和核心测试 |
| M2 稳定看板体验 | 看板与任务生命周期一致 | 拖拽、计时、进程绑定行为可验证 |
| M3 发布准备 | 文档、测试、构建流程可复现 | `build-guide.md` 和 `test-plan.md` 可支持完整验证 |

## 更新规则

- 新能力进入实现前，先写入 `tasks.md`。
- 里程碑方向变化时，同步更新 `decisions.md`。
- 已完成能力不要长期留在 roadmap；移入 `changelog.md` 或相关契约文档。
